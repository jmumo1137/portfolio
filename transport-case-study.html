Transport Management Portal

An enterprise-grade system for automating transport operations, enforcing data integrity, and tracking orders from creation to payment.

  • Role: Full-Stack Developer
  • Type: Enterprise Operations System
  • Stack: React, Node.js, Express, Knex, SQL, REST APIs, Webhooks

Problem Statement

Transport operations are often managed using manual processes, spreadsheets, and disconnected systems. This leads to missing documents, fuel misuse, incorrect mileage records, delayed payments, and poor visibility across the order lifecycle.

The challenge was to build a system that enforces operational discipline, prevents invalid actions, and provides real-time visibility for administrators, dispatchers, and drivers.

Solution Overview

The Transport Management Portal was designed as a lifecycle-driven system where every order moves through predefined states. Each transition is guarded by validations, ensuring that required data and documents are present before progressing.

  • Lifecycle-based order flow
  • Role-based access and actions
  • Strict server-side validations
  • Automated alerts and payment verification

System Architecture

The system follows a modular, API-driven architecture separating concerns between frontend, backend, and external integrations.

  • Frontend (React): dashboards, forms, guarded navigation
  • Backend (Node.js & Express): business logic, validations, APIs
  • Database (SQL): orders, drivers, vehicles, fuel, mileage, payments
  • Webhooks: payment status verification

Order Lifecycle & Validations

Every order follows a strict lifecycle. Transitions are only allowed when system-defined conditions are met. 

Created → Assigned → Loaded → En Route → Delivered → Paid → Closed
  • Orders cannot proceed without assigned driver and vehicle
  • Fuel logging requires receipt, liters, and cost
  • Mileage validation ensures logical odometer readings
  • Proof of Delivery is mandatory before delivery confirmation
  • Orders close only after verified payment via webhook

Key Features

  • Driver and vehicle management
  • Fuel and mileage tracking with integrity checks
  • Waybill and document management
  • Real-time alerts and status updates
  • Automated invoice generation and payment validation

Security & Reliability

  • JWT-based authentication
  • Role-based access control
  • Server-side validation for all critical actions
  • Consistent error handling and logging

Results & Impact

The system significantly reduces manual errors, enforces operational discipline, and provides clear visibility across all transport activities.

  • Improved data accuracy and traceability
  • Reduced disputes around fuel, mileage, and payments
  • Centralized operational reporting
